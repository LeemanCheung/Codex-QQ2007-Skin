## 兼容策略

- 外壳：通过左栏与主区共同祖先定位，不依赖 `#root > div:first-child`。
- 标题栏：运行时标记真实 topbar host，CSS 只作用于该宿主。
- 交互：个人资料、模型和发送按钮通过语义属性及可操作性检查识别。
- 设置：兼容 `role="button"` 和 `role="link"` 的“返回应用”；以当前服务行的真实图标为准。
- 失败保护：无法确认原生应用完整时 `verify` 返回非零退出码。

## 不保证稳定的部分

Codex DOM、类名和 Electron/Chromium 行为没有作为第三方主题 API 对外承诺。官方更新可能导致：

- 容器层级、Tailwind 类或语义标签改变；
- 设置菜单增加、重命名或移除；
- 模型/发送按钮在运行和空闲状态间更换；
- CDP 行为随内置 Chromium 变化。

Chrome DevTools Protocol 官方文档也说明 tip-of-tree 协议会频繁变化且不保证向后兼容。升级后应重新运行完整验收，而不是沿用旧 `verify.json`。
